# 3. Use Cases

## 3.1 System Actors

The system includes the following actors:
- Admin
- Library Staff
- Member
- Guest

---

## 3.2 Use Cases by Actor

### Admin
- Manage users (add, update, delete users)
- Manage system settings
- View system reports
- Monitor system activity

### Library Staff
- Add new books
- Update book information
- Delete book records
- Record borrowing transactions
- Record return transactions
- Search for books
- View overdue books

### Member
- Register account
- Log in to the system
- Search for books
- View book availability
- Borrow books
- View personal borrowing history

### Guest
- Browse available books
- Search for books
- View book details
- Register as a member

---

## 3.3 System Scope Overview

The Library Management System allows different types of users to interact with the system based on their roles. Guests can browse and search for books, while members can register and borrow books. Library staff manage books and transactions, and admins oversee the entire system including reports and user management.
