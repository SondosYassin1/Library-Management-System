# 1. Introduction

## 1.1 Problem Statement
Traditional library systems rely heavily on manual record-keeping, which leads to several issues such as difficulty in organizing books, tracking borrowed and returned items, and maintaining accurate records. This often results in lost books, duplicate data, delayed returns, and inefficient service. Therefore, there is a need for a centralized digital system that improves accuracy, efficiency, and accessibility.

## 1.2 Project Objectives
The main objectives of this system are:
- To provide a centralized platform for managing all library books
- To improve the efficiency of borrowing and returning processes
- To reduce human errors and eliminate duplicate records
- To track book availability and overdue books
- To generate reports that support decision-making

## 1.3 Scope

### In Scope
- Book management (add, update, delete)
- Viewing book availability
- Recording borrowing and return transactions
- Tracking overdue books
- Searching for books (title, author, ISBN)
- Basic reporting system
- Managing user roles (Admin, Staff, Member, Guest)

### Out of Scope
- Online payment systems
- Mobile application
- Integration with external libraries
- AI-based recommendation systems
- Book delivery services

## 1.4 Assumptions & Constraints

### Assumptions
- The system will be used by trained library staff
- Users will access the system through a web browser
- The system will use a centralized database

### Constraints
- Limited development time (graduation project)
- Limited team size
- Fixed technology stack (ASP.NET Core MVC, SQL Server)
- Some advanced features may not be implemented in the first version
